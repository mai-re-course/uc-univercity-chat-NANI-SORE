## Редактирование чата и его участников

# Сценарии использования

**Акторы:** пользователь, администратор, система.

**Цель:** пользователь меняет параметры чата и участников, приглашает участников.

**Предусловия:** пользователь зашёл в приолжение и авторизовался, у него открыт какой-нибудь чат.

**Сценарий:**

1. Пользователь открывает список всех настроек чата и всех его участников, нажав кнопку «шестеренка» в меню.
2. Приложение запрашивает список участников у бекенда методом `GET /chat/users` и доступных для изменения данному пользователю настроек методом `GET /chat/editrights`
3. Бекенд возвращает список пользователей в структуре `users` и список настроек в структуре `options`:
    ~~~json
    {users: [user-data]}
    ~~~

    ~~~json
    {options: [options-data]}
    ~~~

4. Приложение показывает список всех доступных настроек и участников чата.
5. Пользователь меняет необходимые настройки или приглашает новых участников.
6. Пользователь сохраняет изменения.
7. Приложение отправляет изменения на бекенд.
8. Бекенд возвращает ответ успешного изменения параметров чата.
9. Приложение возвращается к открытому чату.

**Результат:**

- Новый изменения вступают в силу сразу после сохранения.
- Администраторам чата видна дата последнего изменения параметров чата.
- В базе данных изменены параметры чата.

**Альтернативные:** 

5.1 {Изменение настроек чата}

5.1.1. Пользователь выбирает и редактирует нужные опции из списка настроек.

5.1.2. После изменения всех нужных пользователю настроек, он может нажать "Сохранить" для сохранения изменений или "Отмена" для их отката, нажатие на любую из них закроет окно настройки чата.

5.2 {Редактирование участников чата}

5.2.1. Пользователь выбирает участника, права/членство которого он хочет изменить, если у него есть привилегии на данную операцию.

5.2.2. Приложение открывает окно с параметрами и кнопками действий над выбранным участником чата. (Его роль в чате, удаление из чата...) 

5.2.3. После изменения всех нужных параметров, он может нажать "Сохранить" для сохранения изменений или "Отмена" для их отката, нажатие на любую из них закроет окно настройки участника.

5.3 {Добавление участников}

5.3.1. Пользователь вводит в поиск или выбирает из списка другого пользователя, которого он хочет добавить.

5.3.2. После выбора возврат на окно настроек
